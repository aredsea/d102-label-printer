# D102 라벨 인쇄 시스템 — 기획서 (PRD)

> 유비샵 `바코드인쇄`를 크롬 확장이 가로채, **자체 인쇄 프로그램**이 크롬 인쇄창
> 없이 Zebra로 직접 인쇄한다. 확장·프로그램 모두 GitHub로 자동업데이트, Win11 x64
> 설치 프로그램으로 둘 다 자동설치.

작성 2026-06 · 결정: **ZPL 네이티브 인쇄** + **설치 시 확장 자동등록(1회 클릭)**

---

## 1. 문제 정의

- 크롬 확장만으로는 인쇄 다이얼로그를 없앨 수 없음(보안). `--kiosk-printing` 우회는
  매장 PC에서 반복 실패(프로세스 병합·설정 취약).
- 근본 해결: **인쇄를 크롬 밖(자체 프로그램)에서** 수행. 확장은 데이터만 넘긴다.

## 2. 솔루션 (아키텍처)

```
[유비샵]  바코드인쇄 클릭
   │  확장이 window.sendBarPrint 가로챔
   ▼
[크롬 확장 — 얇은 호출자]
   ├ 체크된 상품을 infoItemBarPrint.do 파싱(기존 로직 재사용)
   ├ POST http://127.0.0.1:17600/print   { items:[ {barcode,itemName,price,…} ] }
   └ 로딩 오버레이 → 결과 토스트
   ▼
[D102 인쇄 프로그램 — .NET 8 x64, 트레이 상주]
   ├ 로컬 HTTP 서버(127.0.0.1, CORS 허용: ubdstore.ubshop.biz)
   ├ 라벨 템플릿/레이아웃 보유(위치·고정문구·폰트) — 설정 UI에서 편집
   ├ ZPL 생성: 바코드=네이티브 ^BC / 한글·텍스트=Pretendard 비트맵 ^GF
   ├ Zebra로 ZPL raw 전송(Win32 스풀러) → 다이얼로그 없이 인쇄
   └ 설정창(WebView2, 오늘의집 디자인): 라벨 위치편집·프린터 선택·상태
   
[자동업데이트 — 둘 다 GitHub]
   ├ 확장:  GitHub 라이브로더(현행 유지)  aredsea/ubishop-barcode-ext
   └ 프로그램: Velopack(GitHub 릴리스)  aredsea/d102-label-printer

[설치 프로그램 — Win11 x64]
   ├ 프로그램 설치 + 시작프로그램(트레이 상주) 등록
   └ 크롬 확장 레지스트리 자동등록(External Extension) → 크롬에서 "사용" 1회 클릭
```

### 역할 분리 (핵심)
- **확장 = 얇은 호출자**: 유비샵 세션에서 상품 데이터만 수집해 프로그램에 POST.
  (라벨 렌더·위치편집·인쇄 미리보기 전부 제거 → 확장 단순해짐)
- **프로그램 = 렌더·인쇄 담당**: 레이아웃 소유, ZPL 생성, Zebra 직접 인쇄, 설정 UI.

## 3. 사용자 스토리

1. 직원이 유비샵에서 상품 체크 → `바코드인쇄` → **즉시 인쇄**(다이얼로그 0, 로딩 표시).
2. 사장이 라벨 위치/문구를 바꾸려면 트레이의 **D102 인쇄 설정**을 열어 편집·저장.
3. 코드가 개선되면 매장 PC가 **자동 업데이트**(확장=새로고침, 프로그램=재시작 시).
4. 새 PC 세팅: **설치 프로그램 1개 실행** → 프로그램 설치 + 확장 등록, 크롬에서 1회 "사용".

## 4. 구현/테스트 결정

- **인쇄**: ZPL 네이티브. Zebra GX430t = 300dpi(=12 dots/mm, Labelary `12dpmm`).
  - 바코드: `^BCN`(Code128). 텍스트(한글 포함): Pretendard로 1-bit 비트맵 렌더 → `^GFA`.
  - mm→dots = ×11.81. 라벨 60×10mm = 709×118 dots.
- **데이터 계약(확장→프로그램)**: JSON `{ items:[{ barcode, itemName, itemNo, price,
  metal, diameter, weight, category, partner }] }`. 고정문구(브랜드)·레이아웃은 프로그램 보유.
- **검증**: 프린터 없이 **Labelary API**(ZPL→PNG)로 라벨 시각 검증. 실제 인쇄는 매장에서.
- **스택**: .NET 8 (WinForms 트레이 + WebView2 설정창) self-contained x64.
  로컬서버=HttpListener. raw인쇄=OpenPrinter/WritePrinter P/Invoke. 자동업데이트=Velopack.
- **디자인**: 첨부 DESIGN.md(오늘의집) 준수 — 시안 `#35C5F0`, Pretendard, 8/12px 라운드,
  미니멀, **이모지 금지→SVG 아이콘(Lucide)**, 그림자 약하게.

## 5. 범위 외 (이번)

- 유비샵 서버/솔루션 수정. 라벨 양식 신규 디자인(기존 복제). 모바일/웹 버전.
- Web Store 등록(자체 호스팅 유지). 다국어.

## 6. 단계 (검증 체크포인트)

| 단계 | 내용 | 검증 |
|---|---|---|
| **P1** | ZPL 엔진 코어: 레이아웃→ZPL(바코드+한글 비트맵) | Labelary PNG가 사진 라벨과 일치 |
| **P2** | .NET 프로그램 스캐폴드: 로컬서버 + raw인쇄 + 트레이 | 더미 ZPL이 프린터로(매장) / 서버 응답 OK |
| **P3** | 확장 리팩터: 얇은 호출자(데이터 POST)로 축소 | 바코드인쇄→프로그램 수신 확인 |
| **P4** | 설정 UI(WebView2, 오늘의집): 위치편집·프린터·상태 | 위치 저장→ZPL 반영 |
| **P5** | 자동업데이트: Velopack(프로그램) + 확장 로더 | 버전 올리면 자동 갱신 |
| **P6** | 설치 프로그램: 프로그램+확장 자동등록 | 새 PC 1회 설치로 동작 |

> 각 단계 끝에 검증 후 다음 단계. P1은 프린터 없이 가능(최우선).
